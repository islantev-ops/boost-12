## Правки по ревью: check7 summary

### Контекст

Ревью нашло последний остаток класса проблемы «summary не различает охват»:
в `check7` (активное согласие в формах, `src/lib/checks.ts`) ветка «нарушений
нет» была зашита одной фразой на два разных случая — «форм нет, обход полный»
(вывод достоверен) и «форм нет, обход неполный» (форма могла быть на
непросмотренной странице). Это тот же класс инцидента, с которого началась
работа: аудит написал «форм сбора персональных данных не найдено», открыв
5 страниц из 15 (gdpgroup.ru, 2026-07-21) — форма была на неоткрытой странице.

плюс 2 новых теста).

`npx tsc --noEmit` — чисто, без ошибок.

### Правка

В `src/lib/checks.ts`, в `check7`, последняя ветка `summary` заменена с

```ts
: 'Заранее отмеченных чекбоксов не найдено. Сохранение факта согласия требует ручной проверки.',
```

на различение по `pdForms` и `canProveAbsence(s)` (без создания новых
функций — `pdForms`, `canProveAbsence`, `absenceUnknownReason` уже существовали
в области видимости):

```ts
: pdForms === 0
    ? `Форм сбора персональных данных не найдено. ${
        canProveAbsence(s)
          ? 'Сайт обойдён полностью, поэтому вывод достоверен. Сохранение факта согласия всё равно определить нельзя: по внешнему виду сайта оно не видно.'
          : absenceUnknownReason(s)
      }`
    : `Во всех найденных формах сбора персональных данных (${pdForms}) есть чекбокс согласия. Сохранение факта согласия требует ручной проверки: по внешнему виду сайта оно не определяется.`,
```

### Дословные тексты `summary`

Проверено запуском кода напрямую (временный скрипт `verify_check7_tmp.ts` в
корне `product-mvp`, выполнен через `node --import tsx`, затем удалён — в
финальном `git status` отсутствует):

1. **Без форм, обход неполный** (`coverage: { crawled: 300, discovered: 1200,
   skippedByTemplate: 40, skippedByLimit: 860, complete: false,
   stopReason: 'pageLimit' }`, CLEAN_HTML без форм, подвал виден, не SPA):

   > Форм сбора персональных данных не найдено. Обойдено страниц: 300 из 1200
   > найденных на сайте, достигнут потолок обхода. На осмотренных страницах не
   > найдено, но заявлять отсутствие нельзя — часть сайта не просмотрена.

2. **Без форм, обход полный** (тот же HTML, `coverage.complete: true`):

   > Форм сбора персональных данных не найдено. Сайт обойдён полностью, поэтому
   > вывод достоверен. Сохранение факта согласия всё равно определить нельзя: по
   > внешнему виду сайта оно не видно.

3. **Формы есть, во всех есть чекбокс согласия** (форма с `input[type=email]` и
   непомеченным `input[type=checkbox]` рядом с текстом согласия):

   > Во всех найденных формах сбора персональных данных (1) есть чекбокс
   > согласия. Сохранение факта согласия требует ручной проверки: по внешнему
   > виду сайта оно не определяется.

### Подтверждение регресса на ветках с нарушениями

Тем же способом (временный скрипт, до удаления) проверены обе ветки с
нарушениями — текст не изменился относительно кода до правки, так как правка
трогала только последнюю (третью) ветку тернарника:

- **Заранее отмеченный чекбокс согласия**:

  > В формах есть чекбоксы согласия, отмеченные по умолчанию (найдено: 1).
  > Заранее проставленная галочка не является согласием: посетитель не
  > совершал действия.

- **Форма без чекбокса согласия**:

  > Формы собирают персональные данные без чекбокса согласия (найдено форм: 1).
  > Обработка ведётся без согласия в письменной форме.

Кроме того, все 73 ранее зелёных теста (включая проверки других чек-функций)
остались зелёными без единой правки в них — правка ограничена одной веткой
одного тернарника в `check7`.

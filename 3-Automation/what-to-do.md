**Длительность:** ~2.5 часа

## Цели воркшопа

После воркшопа студенты смогут:
- Создавать **скиллы** — переиспользуемые команды, которые запускаются через `/имя`
- Создавать **агентов** — помощников с конкретной ролью и вшитой экспертизой
- Связывать скиллы и агентов в **пайплайн** — многоэтапную автоматизацию одной командой
- Формулировать задачу для агента описанием результата, а не пошаговой инструкцией

> ➡️ **Куда это ведёт.** Скиллы, агенты и пайплайны из этого воркшопа — фундамент. Их продолжение (полноценные команды агентов-разработчиков, чат-бот поддержки, UX-тестирование через агента) — в **воркшопе 5 (5-Agent-Teams)**. Здесь мы приоткрываем тему, дальше углубляем её там.

## Мини-глоссарий

| Термин | Что это |
|--------|---------|
| **frontmatter** | Блок настроек в начале файла между `---` (имя скилла, описание) |
| **TOML** | Простой формат «ключ = значение» для конфигов; у субагентов Codex описание лежит в TOML-файле |
| **Оркестратор** | Управляющий скилл/промпт: раздаёт работу субагентам и собирает их результаты |
| **Спаунить** | Запускать (порождать) субагента |
| **max_depth** | Глубина вложенности субагентов у Codex (по умолчанию 1: оркестратор → воркеры) |
| **sandbox** | Песочница: ограничение, что агенту можно делать (читать/писать/сеть) |
| **reasoning** | Уровень «обдумывания» модели (выше — точнее, но дольше и дороже) |
| **pandoc / headless-Chrome** | Инструменты конвертации (например, Markdown → PDF); агент ставит их сам |

> **Про пути к файлам скиллов и агентов:** механизм одинаковый у обоих агентов, но папки разные. Где это важно, мы помечаем варианты так:
>
> - **🟠 Claude Code:** путь/команда для Claude Code
> - **🔵 Codex:** путь/команда для Codex
>
> Сами промпты ниже нейтральны — пиши их любому агенту. Различаются только места, где лежат готовые файлы.

### Задание 10: Создаём скилл — поиск новостей + написание постов (15 минут)

**Модель:** 🟠 Claude Opus · 🔵 Codex reasoning high

**Зачем:** Скилл может содержать вшитые знания (карту тем канала), запускать инструменты (поиск в интернете) и оркестрировать агентов. Всё в одной команде.

```
В папке 3-Automation/data/block-2-content/telegram-posts/ лежат 53 поста моего канала @data_secrets.
У меня уже есть агент post-writer, который пишет посты в стиле канала.
Хочу скилл с именем create-posts, который:
1. Анализирует мои посты и понимает, о чём я пишу
2. Ищет свежие инфоповоды по моим темам в интернете
3. Запускает агента post-writer на каждый найденный инфоповод
Результат работы скилла — все посты в один файл 3-Automation/results/posts-[дата].md.
Только создай скилл, не запускай его.
```

> **Про поиск в интернете:** у обоих агентов поиск встроен. Здесь нужны **свежие** инфоповоды, поэтому поиск должен быть живым, а не из кэша.
> - **🟠 Claude Code:** доступен инструмент WebSearch — ищет вживую.
> - **🔵 Codex:** web search по умолчанию в режиме `cached` (из кэша) — для свежих новостей **обязательно** переключи на живой. Пошагово:
>   1. Разовый запуск: добавь флаг `--search` к команде запуска Codex.
>   2. Либо навсегда: открой `~/.codex/config.toml` и пропиши `web_search="live"`.
>   3. Проверь, что включилось: попроси агента «найди новость за сегодня» — если даёт сегодняшнюю дату, живой поиск работает.
>
>   Иначе посты будут по устаревшим инфоповодам — это противоречит самой идее «свежий контент».

> **Как это устроено у разных агентов:**
> - **🟠 Claude Code:** скилл `create-posts` ищет инфоповоды и вызывает агента `post-writer` на каждый — скилл + агенты.
> - **🔵 Codex:** оркестратор-промпт (скилл) сам ищет инфоповоды, спаунит субагентов `post-writer` параллельно — по одному на инфоповод — и собирает посты в один файл. Вся оркестрация — на уровне скилла; сам `post-writer` (лист) ничего не спаунит (`max_depth=1`, и этого хватает: оркестратор → воркеры). «Явный запрос» на субагентов = твой вызов `/create-posts` — отдельно ничего активировать не нужно.

> **Как поставить скилл из такого репозитория (чтобы агент сделал это сам, а не ответил «сделай сам»):** проси не «установи скилл», а конкретное действие с файлами. Надёжная формулировка:
> ```
> Склонируй репозиторий [URL] во временную папку, найди в нём папку скилла [имя]
> и скопируй её целиком в .claude/skills/ моего проекта (для Codex — в .agents/skills/).
> Сделай это сам, файлы за меня не оставляй. После — покажи, что скилл на месте.
> ```
> Принцип: установка скилла = это копирование папки скилла в твою папку скиллов. Если агент предлагает «сделай вручную» — повтори запрос как явную команду «склонируй и скопируй файлы сам».
>
> *Точное число скиллов в этих каталогах меняется от обновления к обновлению — ориентируйся на порядок («десятки»), а не на конкретную цифру.*
>
> Эти каталоги собраны в формате Claude Code (`.claude/skills`, `.claude/agents`). Идеи и инструкции из них переносимы и на Codex — но пути и формат файлов придётся поправить под Codex (`.agents/skills`, `.codex/agents/*.toml`). У Codex есть и свой каталог расширений — система Plugins и официальный маркетплейс «Curated by OpenAI» (команда `codex /plugins`; плагин в Codex — это манифест `.codex-plugin/plugin.json`).

## Что мы построили

Пример структуры для **🟠 Claude Code**:

```
.claude/
├── skills/
│   ├── create-contract/SKILL.md   ←  генератор договоров (задания 2-3)
│   ├── batch-contracts/SKILL.md   ←  пайплайн: несколько договоров параллельно (задания 5-6)
│   ├── create-posts/SKILL.md      ←  поиск новостей + посты через агента (задания 10-11)
│   ├── adapt-resume/SKILL.md      ←  практика, вариант А: резюме → 5 адаптаций
│   └── score-candidates/SKILL.md  ←  практика, вариант Б: 5 кандидатов × 2 вакансии
└── agents/
    ├── contract-generator.md      ←  генератор договоров (задание 4)
    ├── post-writer.md             ←  копирайтер (задания 8-9)
    ├── resume-adapter.md          ←  практика, вариант А: адаптация резюме
    └── candidate-scorer.md        ←  практика, вариант Б: скоринг кандидатов
```

> **🔵 Codex:** аналог — скиллы в `.agents/skills/<имя>/SKILL.md` (личные — `~/.agents/skills/`), субагенты в `.codex/agents/<имя>.toml` (TOML вместо `.md`). Та же логика «скилл оркестрирует агентов», но субагенты запускаются по явному запросу и не переписываются между собой.

# Формат скиллов и агентов

> Этот файл описывает формат скиллов и агентов для **обоих** AI-агентов — Claude Code и Codex. Механизм одинаковый, отличаются только пути к файлам и формат описания агента.
>
> - **🟠 Claude Code:** читает правила проекта из этого файла (`CLAUDE.md`).
> - **🔵 Codex:** читает правила проекта из `AGENTS.md` (рядом лежит файл с тем же содержимым).

---

## Скиллы

Скилл — это сохранённая инструкция, которую пользователь вызывает через `/имя-скилла` в новом чате (либо упоминанием имени скилла).

Обязательно:
- Папка скиллов в корне проекта
- Внутри — подпапка с именем скилла (например `create-contract`)
- Внутри подпапки — файл с именем именно `SKILL.md` (большими буквами)

**Путь:**

> **🟠 Claude Code:** `.claude/skills/имя-скилла/SKILL.md`
> Пример: `.claude/skills/create-contract/SKILL.md`
>
> **🔵 Codex:** `.agents/skills/имя-скилла/SKILL.md` (репозиторий) или `$HOME/.agents/skills/имя-скилла/SKILL.md` (личные)
> Пример: `.agents/skills/create-contract/SKILL.md`
> *Путь скиллов Codex — `.agents/skills/` в проекте, `~/.agents/skills/` для личных. У субагентов Codex папка другая — `.codex/agents/*.toml` (см. ниже): скиллы и субагенты — разные механизмы, поэтому и папки разные — так и задумано.* <!-- TODO: сверить путь с докой Codex --> <!-- правка: FIXLIST 🟡 -->

**Формат файла** (почти одинаковый у обоих — YAML-frontmatter `name` + `description`, дальше инструкции):
```markdown
---
name: имя-скилла
description: Что делает скилл (одно предложение)
---

[Инструкции для агента: что делать при вызове этого скилла]
```

## Агенты

Агент — это помощник с ролью, которого запускают из скиллов.

**Путь и формат:**

> **🟠 Claude Code:** `.claude/agents/имя-агента.md` — Markdown с frontmatter.
> Пример: `.claude/agents/post-writer.md`
> ```markdown
> ---
> name: имя-агента
> description: Роль агента (одно предложение)
> ---
>
> [Системный промпт: роль, экспертиза, формат работы]
> ```
>
> **🔵 Codex:** `.codex/agents/имя-агента.toml` (проектные) или `~/.codex/agents/имя-агента.toml` (личные) — формат TOML.
> Пример: `.codex/agents/post-writer.toml`
> ```toml
> name = "имя-агента"
> description = "Роль агента (одно предложение)"
> developer_instructions = """
> Системный промпт: роль, экспертиза, формат работы
> """
> # опционально: model, reasoning, sandbox
> ```

**Важно про субагентов Codex:** субагенты запускаются **только по явному запросу пользователя** и работают **параллельно**, но **не общаются между собой напрямую** — нет аналога «агент пишет другому агенту и ждёт ответ». Архитектура иерархическая: оркестратор (скилл / промпт) спаунит субагентов, ждёт всех и сам собирает результаты. По умолчанию `max_depth=1` — без вложенных команд.

## Файл правил проекта

Кроме скиллов и агентов, у проекта есть главный файл правил, который агент читает автоматически:

> **🟠 Claude Code:** `CLAUDE.md` (этот файл). Глобальный — `~/.claude/CLAUDE.md`.
> **🔵 Codex:** `AGENTS.md` в корне проекта. Глобальный — `~/.codex/AGENTS.md`; во вложенных папках можно класть свои `AGENTS.md`. У суммарного размера подхватываемых файлов правил есть лимит (настраивается; по умолчанию небольшой) — держи правила компактными.
